## 一、Claude Code 的四项基本功

Claude Code 的核心能力可以归纳为 4 类：

```
读写文件   ←  搜索代码   ←  运行命令   ←  操作 Git
   │            │            │            │
   ▼            ▼            ▼            ▼
创建/修改代码   找到相关代码   构建/测试/调试   提交/推送/合并
```

---

## 二、文件操作（最常用）

### 2.1 读取文件

Claude Code 会自动读取你项目中打开的文件。你也可以直接说：

```
请查看 src/main/java/com/example/HelloController.java
```

### 2.2 创建文件

```
帮我创建一个 UserService.java，包含用户的增删改查方法
```

Claude 会生成完整代码并创建文件。

### 2.3 修改文件

```
在 UserService 中加入一个根据邮箱查找用户的方法
```

Claude 会定位到目标文件，找到准确位置进行修改。

### 2.4 删除文件

```
删除 src/test/Test.java 这个文件
```

> ⚠️ 需要你确认后才会执行删除操作。

## 三、代码搜索

当你面对一个大项目，不知道某个功能写在哪里时：

```
搜索项目中所有用到 @Autowired 的地方
```

### 搜索方式

| 方式 | 适合场景 | 示例指令 |
|------|---------|---------|
| 关键词搜索 | 找特定单词 | "搜一下项目中哪里调用了 userService" |
| 文件名搜索 | 找特定文件 | "找到项目中所有的 Controller 类" |
| 语义搜索 | 找功能实现 | "用户登录的逻辑在哪里实现的？" |

---

## 四、运行命令

Claude Code 可以在终端执行命令，但**所有命令都需要你确认后才运行**。

```
# 你可以这样说
帮我查看项目依赖
安装 express 包
运行测试用例
查看 Git 提交历史
```

### 权限机制

```
Claude: 需要运行 "npm install express"（Y/n）？
       ↑ 你必须输入 Y 确认才会执行
```

**安全原则**：Claude 不会未经你确认就运行任何命令。

---

## 五、Git 操作

Claude Code 能帮你完成大部分 Git 操作：

```
# 日常操作
查看当前修改了哪些文件
帮我提交代码，提交信息写 "feat: 添加用户注册功能"
推送到远程仓库

# 高级操作
帮我创建一个新分支 feature-login
看看这个分支和 master 有什么区别
帮我合并冲突
```

### 提交信息规范

Claude 会按照常规提交规范生成提交信息：

```
feat: 新功能
fix: 修复bug
docs: 文档更新
refactor: 重构
test: 测试相关
chore: 构建/工具相关
```

## 六、斜杠命令（快捷操作）

在交互模式下，输入 `/` 可以看到所有可用命令：

| 命令 | 功能 | 新手推荐度 |
|------|------|-----------|
| `/help` | 查看所有命令 | ⭐⭐⭐ |
| `/clear` | 清空对话记录 | ⭐⭐⭐ |
| `/cost` | 查看 token 消耗 | ⭐⭐ |
| `/review` | 审查当前代码变更 | ⭐⭐⭐（很实用） |
| `/fix` | 自动修复问题 | ⭐⭐ |
| `/explain` | 解释选中的代码 | ⭐⭐⭐（学习用） |
| `/init` | 初始化 CLAUDE.md | ⭐⭐ |
| `/compact` | 压缩对话节省上下文 | ⭐⭐⭐（长对话必用） |

### 使用示例

```
/review          → Claude 会审查你当前的改动
/explain         → 解释光标所在的代码
/compact         → 对话太长时压缩一下继续聊
```

## 七、Memory（记忆功能）

Claude Code 可以记住你的偏好，下次会话继续使用。

### 存储的内容类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `user` | 你的身份和偏好 | "我是 Java 后端开发者" |
| `feedback` | 给你的工作反馈 | "代码要写中文注释" |
| `project` | 项目相关信息 | "这个项目用 Spring Boot 3.2" |
| `reference` | 外部资源链接 | "API 文档在 xxx 地址" |

### 使用示例

```
记住我是 Java 后端开发者，代码注释用中文
```

之后 Claude 会一直遵循这个偏好。

## 八、Skill（技能系统）

# 当你需要代码审查时
/code-review

# 当你需要配置更新时

## 九、理解权限模型

Claude Code 操作文件时，会经过这样一个流程：

```
你提出需求
    │
    ▼
Claude 理解并规划方案
    │
    ▼
Claude 尝试执行操作（如创建文件）
    │
    ▼
⚠️ 系统弹出权限确认（Y/n）
    │
    ▼
你确认 → 操作执行
```

**新手常问**：
- **Q：为什么有时候 Claude 不动了？**
  A：它在等你确认权限，看终端提示输入 Y
- **Q：可以不每次都确认吗？**
  A：可以在 settings.json 中配置权限模式
