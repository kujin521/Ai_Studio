# Claude Code 实战技巧与最佳实践

> 从会用→用好，这些技巧能让效率翻倍。

---

## 一、高效提问的 4 个原则

### 原则 1：明确上下文

```
❌ 不推荐："帮我改一下这个"
✅ 推荐："在 UserController 的 getUserById 方法中，把返回状态码从 200 改成 201"
```

### 原则 2：一次一件事

```
❌ 不推荐："帮我创建用户模块、加权限、写测试、部署到服务器"
✅ 推荐：分步进行
   第1步："帮我创建一个 UserController，包含 CRUD 接口"
   第2步："给 UserController 加上 @Valid 参数校验"
   第3步："为 UserService 写单元测试"
```

### 原则 3：给出代码风格约束

```
"帮我写一个分页查询方法，代码风格和项目中已有的保持一致"
"请用 Lombok 简化实体类"
"注释用中文，方法要写 JavaDoc"
```

### 原则 4：先问再看

```
"在修改之前，先告诉我你打算怎么改"
```

## 二、项目初始化最佳实践

### 第一步：初始化 CLAUDE.md

新项目第一次启动 Claude Code 后：

```bash
/init
```

这个命令会分析项目结构，自动生成 `CLAUDE.md` 文件。后续 Claude 会始终参考这个文件中的规范。

### 第二步：记录项目记忆

```
记住：这个项目使用 Spring Boot 3.2 + JDK 17，controller 统一返回 Result 封装类
```

### 第三步：配置权限（可选）

不想每次都确认？可以设置更宽松的权限模式：

```
/update-config
```

## 三、长对话管理

当对话变长后，Claude 的回复质量可能下降。这时：

### 使用 /compact

```
/compact
```

作用：压缩对话历史，保留关键信息，释放上下文空间。

### 什么时候该 compact？

| 信号 | 说明 |
|------|------|
| Claude 开始忘记之前说过的内容 | 对话已较长 |
| 回复变得不准确或丢失细节 | 上下文过载 |
| 完成了某个大功能 | 准备开始新任务 |
| 看到 /cost 消耗很高 | token 快用完了 |

### 何时该开启新会话

```
对话出现以下情况之一，建议退出重开：
1. 完成了完整的任务模块
2. 话题已经完全转换
3. /compact 后效果仍不佳
```

## 四、代码审查工作流

这是平时最常用的高效工作流：

```
# 第1步：写完代码后审查
/review
→ Claude 会检查代码中的潜在问题

# 第2步：如果需要修复
/fix
→ Claude 自动修复发现的问题

# 第3步：提交代码
"帮我提交，提交信息用 'fix: 修复空指针异常'"
```

### 常规审查流程

```
1. 修改代码 → git add
2. claude
3. /review  (审查改动)
4. 确认问题 → 手动或 /fix 修复
5. 提交
```

## 五、理解 CLAUDE.md

每个项目根目录下的 `CLAUDE.md` 是指引文件：

```
# CLAUDE.md

## 项目概况
告诉 Claude 这个项目是做什么的

## 开发命令
构建、测试、运行命令清单

## 架构说明
项目的核心架构和约定
```

### 为什么要维护它？

- **保持一致性**：每次 Claude 启动都会读取它
- **减少重复说明**：不需要每次都重新告诉 Claude 项目信息
- **团队协作**：所有开发者用同一份规范

### 需要写什么？

```
✅ 应该写的内容：
  - 构建命令（mvn clean install）
  - 运行测试命令
  - 项目架构概述
  - 代码规范约定

❌ 不需要写的内容：
  - "要写好错误处理"（太泛）
  - 每个文件的完整结构（可以通过搜索得知）
  - 通用的开发规范
```

## 六、常见问题诊断

### 问题 1：Claude 无法连接

```
现象：启动时提示网络错误
解决：
  1. 检查网络连接
  2. 检查 API Key 是否有效
  3. 检查是否需要配置代理
```

### 问题 2：命令被拒绝

```
现象：Claude 说"没有权限执行此操作"
解决：
  这是正常的安全机制，检查权限设置或重新确认操作
```

### 问题 3：对话突然中断

```
现象：终端卡住或没有响应
解决：
  按 Ctrl+C 退出 → 重新启动 → 用 claude --resume 恢复
```

### 问题 4：生成代码不符合要求

```
现象：生成的代码和预期不一样
解决：
  用 /compact 压缩对话 → 更详细地描述需求 → 重新尝试
```

## 八、高效开发工作流（完整示例）

```
=== 日常开发循环 ===

1. claude           ← 启动
2. "帮我实现用户登录功能"  ← 提需求
3. 审查 Claude 生成的代码
4. /review          ← 代码审查
5. "帮我提交代码"   ← Git 提交
6. /compact         ← 清理上下文
7. 继续下一步需求...

=== 遇到问题时 ===

1. "运行测试报错了"  ← 粘贴错误信息
2. Claude 分析原因
3. "帮我把这个修好"
4. /review          ← 确认修复没问题
5. "提交修复"       ← Git 提交
```
